# v12 更新说明

1. 批量导入容量增大：默认单次请求上限 `MAX_UPLOAD_MB=512`，批量最多 `MAX_BATCH_FILES=500` 个文件。前端会按 20 个文件一组分批上传，避免一次性上传过大导致中断。
2. 文章删除按钮缩小并移动到右侧：文章投喂页新增“小号删除当前结果”按钮；每篇文章卡片右侧也有小号“删除”。删除前会要求确认。
3. 自动保存增强：新增病例保存到 `data/user_cases.json`，文章保存到 `data/articles.json`，删除病例保存到 `data/deleted_cases.json`。写入采用临时文件替换方式，降低异常中断导致 JSON 损坏的概率。
4. 新增 `/api/storage/status` 可查看保存路径、文章数量、新增病例数量、上传限制。

注意：如果你把项目重新解压覆盖旧文件夹，旧的 `data/user_cases.json` 和 `data/articles.json` 可能被覆盖。需要长期保留投喂内容时，请备份整个 `data` 文件夹，或只替换代码文件，不覆盖 `data` 目录。
