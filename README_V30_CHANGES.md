# v30 更新说明

本版针对 v29 的四个问题集中修复：

1. API 配置现在可以真正清除后端保存的 Key。此前“清空 API”只清了浏览器 localStorage，后端仍可能读取旧 Key，导致继续调用失败并反复输出兜底回答。
2. 取消左侧“吉祥物设置”按钮。现在直接点击页面中的吉祥物图片即可上传新图片替换。
3. 存储系统改为 persistent-only：运行时只读取 `C:\Users\你的用户名\.uscc_scc_flask_data`，不再自动导入项目自带的 `data/articles.json` / `data/user_cases.json`，避免旧版固定 70 篇文章反复恢复。
4. 回答速度优化：普通追问不再强制检索病例/文献；API 连接测试和对话调用设置了更短超时；本地兜底回答不再只重复一句固定提示。

如需彻底检查当前存储路径，访问：

```text
http://127.0.0.1:5000/api/storage/status
```

如果你之前的旧持久目录里已经残留了 70 篇文章，可在“投喂文章”页面点击“一键删除”清空。v30 删除后不会再从项目自带文件恢复。
