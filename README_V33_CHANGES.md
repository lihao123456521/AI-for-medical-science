# V33 更新说明

1. API Key 已记住配置增强
   - 支持保存多个 API 配置历史。
   - 点击“调用已记住配置”时，如果有多个配置，会弹出选择窗口。
   - 选择后会把该配置恢复为当前后端配置，后续对话直接调用。

2. 文章图片分析增强
   - 投喂 DOCX/PDF 时会继续提取图片。
   - 若后端已有可用多模态 API 配置，会把“文章正文节选 + 图片”一起交给模型分析。
   - 保存到文章库中的内容会包含“文章图片与正文联合分析”，后续回答可参考。

3. 回答速度与知识库快速记忆
   - 新增 knowledge_digest.json 缓存。
   - 普通问答不再强制全文检索病例和文献，但会把知识库概要交给模型，帮助更快理解本地数据库构成。
   - 只有初次患者分析或医生明确要求病例/文献/治疗讨论时，才进行更重的检索。

4. 保存机制延续 V32
   - 病例、文章、删除记录继续保存到用户目录。
   - library_state.json 继续作为统一快照，用于版本升级或 JSON 文件异常时恢复。
