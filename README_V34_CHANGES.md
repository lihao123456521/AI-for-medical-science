# V34 更新说明

本版在 v33 基础上修复三项问题：

1. API 历史配置清理
   - 未连接成功的 API 配置不会再保存为可调用配置。
   - 旧版本中未成功连接或缺少关键信息的配置会自动从历史列表中过滤。
   - 历史配置缺少编号的问题已修复：成功保存的新配置会带稳定 config_id。

2. API 历史配置调用
   - “调用已记住配置”现在只显示成功连接过的配置。
   - 如果有多个成功配置，会弹窗选择。
   - 选择后会切换到该配置并作为后端当前配置。

3. 清空文章库
   - 打包内 `data/articles.json` 已置空。
   - 首次运行 v34 时，会清空持久目录中的文章库，并写入清空标记，防止旧的 70 篇文章重启后恢复。
   - 后续重新投喂的文章会正常保存。
