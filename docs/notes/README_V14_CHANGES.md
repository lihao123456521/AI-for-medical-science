# v14 更新说明

## 1. 数据持久化机制

从 v14 开始，医生投喂的病例、文章、删除记录和上传文件默认保存到用户目录：

```text
C:\Users\你的用户名\.uscc_scc_flask_data
```

对应文件：

```text
user_cases.json
articles.json
deleted_cases.json
uploads/
candidate_batches/
```

以后更新前端界面或替换项目代码时，只要不删除这个目录，之前投喂的数据不会丢失。

如果需要指定保存位置，可设置环境变量：

```powershell
setx USCC_DATA_DIR "D:\USCC_DATA"
```

首次运行 v14 时，程序会尝试从当前项目及同级旧项目的 `data/user_cases.json`、`data/articles.json`、`data/deleted_cases.json` 自动迁移旧数据。

## 2. 病例知识库按钮调整

“删除当前结果”按钮已缩小并移动到知识库页吉祥物图标左侧。

## 3. 回答逻辑调整

相似病例与“治疗/转归对照”合并输出，不再分成两个重复模块。默认展示有借鉴价值的相似病例，最多 4 个；文献卡片最多 4 个。

## 4. 文献理解

回答会优先把文章内容转化为中文建议，不照抄文献原文。详细文章仍通过下方文献卡片跳转查看。

## 5. 大型病例表匹配

在对话框上传包含大量病例的 Excel/CSV 后，系统会识别候选病例。医生继续输入患者年龄、姓名、诊断、病理、住院号等信息后，系统会在回答下方跳出候选病例卡片，供医生点击确认。
