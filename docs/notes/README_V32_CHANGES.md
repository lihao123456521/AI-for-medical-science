# V32 更新说明

本次相对 v31 的主要调整：

1. **固定项目头像**
   - 全站吉祥物/头像已统一替换为用户指定图片。
   - 取消前端上传更换头像机制，后端对应接口也改为固定返回该头像。

2. **回答样式优化**
   - 后端会自动清理模型回答中的“好的，医生”等机械化开头。
   - 自动去除段首 `###` 这类 Markdown 标题前缀，显示更自然。

3. **病例与文章保存机制增强**
   - 新增 `library_state.json` 统一快照。
   - 保存病例、文章、删除病例后，都会同步写入快照。
   - 启动时若单独 JSON 缺失或损坏，会尝试从快照恢复，降低“更新前端后数据丢失”的风险。

4. **API 配置记忆增强**
   - API 配置继续保存到后端 `api_config.json`。
   - 新增最近配置历史 `api_config_history.json`。
   - 前端新增“调用已记住配置”按钮，重新打开网页后可一键调用后端记住的配置。
   - 打开配置弹窗时，会自动显示当前后端已记住的 provider/model/base_url/key 掩码信息。
